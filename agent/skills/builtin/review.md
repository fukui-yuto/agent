---
name: review
description: 現在のgitの差分をコードレビューする
---
`git diff` または `git diff --staged` で現在の変更を取得し、以下の観点でコードレビューを行ってください。

レビュー観点:
- **バグ・論理エラー**: 明らかな誤りや問題のある実装がないか
- **セキュリティ**: SQLインジェクション・XSS・機密情報のハードコード等がないか
- **パフォーマンス**: 不要な処理・N+1問題・重いループ等がないか
- **可読性**: 変数名・関数名が適切か、コードが理解しやすいか
- **テスト**: テストが必要な変更に対してテストが書かれているか

各問題点は重要度（高/中/低）と具体的な改善案を添えて報告してください。
